## The decision

Build the **self-review heartbeat first** (tier 1), the **external dispute/oracle feed
second** (tier 2) — the two-tier proposal — and treat the **deterministic/published
selection rule as a binding constraint** on every autonomous source.

## Why (the panel converged on one argument)

**The binding constraint decides the ordering** (V1, sharpest): the sourcer holds agenda
control — flood, omit, time-to-swing — so "an operator chose this one" makes the 1.1
daemon "just an automated delivery truck for a human-curated queue; autonomy is theater."
A deterministic + outsider-reproducible selection rule is the only thing that closes that
gap, and it is **trivially satisfiable for self-review** (trigger on a new commit/diff or
fixed cadence, reproducible from public history) but **the hard, unsolved problem for the
external feed** (the ripeness/selection filter is precisely the cherry-pick surface, plus
a new external trust/availability dependency). Sound sequence: prove the autonomous
*mechanism* on the provably-clean, zero-dependency source, harden the daemon against that
easy case, *then* attack the hard non-cherry-pickable selection rule on a battle-tested
harness. Doing the hard trust problem first stacks two risks at once.

**Round-48 guidance is dispositive on the value objection** (all three): "existence, not
economics." The navel-gazing / "nobody pays $QRM to grade its own homework" risk is real
but it is an *economic-value* concern 1.2 explicitly defers. Self-review proves autonomous
sourcing *exists* with zero new trust surface, is **demonstrably not a rubber stamp** (V2
caught real bugs in self-reviews at rounds 44, 45, 49, 50), and **directly unlocks Phase
1.3** (self-improvement). V3: "internal hygiene must work before external value can be
reliably adjudicated — a panel whose own process is sloppy cannot credibly resolve
external disputes."

**Governance-first rejected** (V1, V3): the proposer is still us, so it is a weaker cousin
of self-review with none of the 1.3 payoff.

## Binding constraint (affirmed 3/3) — carry forward

> The selection rule for ANY autonomous ballot source must be **deterministic and
> published** — reproducible by an outsider from public inputs — so no human (including
> the operator) and no hidden process can hand-pick, cherry-pick, or reorder ballots.
> "An operator chose this one" fails autonomy even if the convening is automated.

This binds tier 1 (the self-review trigger must be reproducible from public history) and,
harder, tier 2 (the external-feed selection rule must be non-cherry-pickable before it
ships).

## Status / next
