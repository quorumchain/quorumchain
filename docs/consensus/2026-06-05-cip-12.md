## 1. Problem

[[CIP-9]]'s `PanelStateReceipt` recorded only `{validators, size}` — *who* was on the panel
and *how many*, but nothing about **how correlated** they were. NI-9a requires every
CONSENSUS record to carry the panel-diversity state that produced it precisely so a
captured- or monocultural-era record is later identifiable; a receipt that cannot express
shared-foundation correlation cannot fully satisfy that intent. The panel chose
(3/3 unanimous, ballot `e87521e46cb67d52…`, log entry 234) to graduate the **correlation
receipt** first — over the calibration and forking graduations — on a dependency argument:
both of those downstream features need panel-composition already on the record.

## 2. Spec (exactly what was ratified)

- **§1 schema extension (additive).** `PanelStateReceipt` gains a per-validator
  `composition` (provider / model-lineage descriptor) and a panel-level `correlationBand`.
  Both are **descriptive-only**: no field feeds back into `status`, `standing`, `verdict`,
  or `panelVotes`.
- **`correlationBand="UNKNOWN until probes ship"`.** The current local pipeline does not run
  correlation probes, so the band is honestly `UNKNOWN` rather than a fabricated number — a
  forward-compatible placeholder, not an overclaim.
- **Status honesty.** CIP-12 ships the *receipt* (observable panel composition), not a
  smarter-than-the-models capability; the accuracy-not-popularity limit from [[CIP-9]] is
  preserved, not oversold.

## 3. Invariants introduced — NI-12a … NI-12i

The nine invariants the consult named, carried faithfully into the ratified draft:

- **NI-12a** — composition is **computed, not assigned** (no editorial labelling of a panel).
- **NI-12b** — **descriptive-not-dispositive**: the receipt has **no verdict / standing /
  status effect**; it may never feed back into the computed record. *(The crux gate.)*
- **NI-12c** — **provenance-frozen / append-only**: a recorded composition is never silently
  rewritten.
- **NI-12d** — **no popularity / reputation signal** leaks through `composition` or
  `correlationBand`.
- **NI-12e** — **anti-Orwell**: correlated foundations are **discountable, not discardable**;
  no automatic discount, no silent erasure.
- **NI-12f** — **no false independence**: a shared foundation raises the correlation band
  (ELEVATED), coherent with [[CIP-11]] NI-11b / [[CIP-10]] same-family logic.
- **NI-12g** — **no silent omission**: a panel's composition is always recorded, never
  dropped.
- **NI-12h** — **privacy-minimality**: only the minimum provenance needed for the receipt.
- **NI-12i** — **schema-forward-compatible**: the calibration and forking graduations can
  consume this receipt later without a migration.

## 4. Compatibility / migration

Append-only / forward-only preserved: the extension is additive to the receipt shape and
changes no computed field, so existing convenings project unchanged. CIP-12 is the read-path
half of NI-9a's diversity-state intent; it touches no T0 frozen-unamendable core item
(append-only history, diversity-as-principle, separation-of-powers, frozen tier-assignment).
