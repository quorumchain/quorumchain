## 1. Problem

[[CIP-13]] NI-13e made re-adjudication **anchor-gated**, but at v0.1 the anchor was an
unverified `newAnchor: boolean` — the supersede *asserted* it cited new anchored evidence and
the gate trusted the assertion. That is exactly the gap [[CIP-10]] / [[CIP-11]] / [[CIP-2]]
exist to close: a self-declared "I have an anchor" is not an anchor. The gate needed to verify
a **real** anchor of demonstrated provenance before a head moves.

## 3. Spec (exactly what was ratified)

- **Typed anchor object** — `{anchorType` (must be in the [[CIP-4]] frozen admissible-type
  policy)`, provenanceClass: STRUCTURED|DOCUMENT, issuer, contentHash` (sha256 of the EXACT
  cited artifact)`, citedAssertion` (the specific supersede claim it supports)`, asOf,
  signature?, issuerKey?, provenanceTrace[]}`.
- **Admissible-type taxonomy names real sources** — Chainlink (OCR aggregate signature +
  on-chain round), Pyth (Wormhole VAA), UMA Optimistic Oracle (dispute-bonded, a distinct
  economic-dispute family), API3 / Chronicle / RedStone / Tellor, zkTLS / TLSNotary (bridges
  class-2 documents into verifiable), OpenTimestamps / RFC-3161 (time only), signed
  institutional releases (class-1 documents), unsigned news/papers (class-2).
- **The family rule** — a decentralized oracle's internal node diversity is **NOT** anchor
  diversity: one Chainlink feed = **one family** per [[CIP-11]] NI-11b regardless of node
  count. `requiredFamilies >= 2` needs *different* networks with distinct data sourcing, and
  the pre-quorum check **collapses** any shared upstream venues via `provenanceTrace`
  (Chainlink + Pyth both reading Binance/Coinbase = one family). A signature proves the oracle
  *said* X, not that X is *true*.
- **`promotable()` change** — NORMATIVE still needs no anchor; EMPIRICAL_LIVE / SETTLED promote
  iff `structurallyAdmissible(anchors)` yields `independentFamilies >= requiredFamiliesFor(type)`.

## 4. Invariants introduced — NI-15a … NI-15g

- **NI-15a** — **claim-bound content hash.** `contentHash` binds the **real cited artifact +
  citedAssertion**, not a free-floating digest.
- **NI-15b** — **halt-over-degrade.** A structural pass is labelled *"anchor-structurally-
  admissible, content-verification-pending"* and is **NEVER** content-confirmed / RESOLVED-as-
  truth. Such a supersede is admitted to content review (`reviewAdmissible`) but **does not
  promote the head** pre-testnet (`contentConfirmed` is set only by the deferred content layer).
- **NI-15c** — **strictly tighter than `newAnchor` on every axis** (the new gate never admits
  something the old boolean would have rejected).
- **NI-15d** — **class-1-first is deferral-not-denial**: class-2 (unsigned news/papers) is
  **logged, not dropped**; no neutrality ranking, no silent suppression.
- **NI-15e** — **anchor-set commitment bound into the hash.** A ballot's canonical
  `anchorCommitment` (a deterministic, replay-stable commitment over the anchor SET) enters its
  ballot hash via the same optional-append discipline as [[CIP-14]]'s type binding, so a
  **post-vote anchor swap fails** verification.
- **NI-15f** — **surfacing-monotonicity**: an anchor must surface **after** the head it
  re-adjudicates sealed (the asOf monotonicity gate).
- **NI-15g** — supporting structural-admissibility / family-independence accounting used by the
  gate (the per-type `requiredFamiliesFor` floor that keeps RESOLVED reachable, NI-11g liveness).

## 5. Compatibility / migration

Append-only / forward-only preserved: anchors attach to a *new* superseding ballot and the
anchor-set commitment is bound by optional-append (NI-15e), so existing signed ballots verify
unchanged. CIP-15 strictly **tightens** the CIP-13 NI-13e gate; it touches no T0
frozen-unamendable core item. The admissible-type policy itself lives under the [[CIP-4]]
frozen policy surface.
