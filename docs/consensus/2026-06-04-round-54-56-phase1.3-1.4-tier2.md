On its first live run, the self-improvement gate refused to approve its own code — the
strongest evidence it is not a rubber stamp. Two real findings:

1. **V1 — the gate did not gate.** `run-self-improve.ts` only `console.log`ged the
   verdict and `main()` returned for every outcome, so the process exited 0 whether
   APPROVED or REJECTED. A scheduler/CI couldn't act on it — "blocks" (claimed) vs "prints
   a label" (actual).
2. **V2 — the gate trusted the mutable `queue/done/<id>.json`** instead of recomputing
   from the tamper-evident log. An edited/stale record could launder `ratified:SOUND` into
   APPROVED — the exact laundering `feed.ts` already avoids.

### Fixes (TDD) — commit `bcaf720`
- `gateForBallot(feed, ballotHash)`: gate on the RECOMPUTED feed (verify chain + ratify
  over signed votes + pinned keyring), keyed by HEAD's review ballot hash. A broken chain
  or a missing convening → INCONCLUSIVE; no stored record is trusted.
- `exitCodeFor(gate)`: non-zero unless APPROVED; `run-self-improve` sets `process.exitCode`
  so the cycle blocks mechanically.
- Cleaned + gitignored the `.qrm_*.txt` rtk diff-capture artifacts a validator's `git
  show` leaves behind (V1 noted).

## Round 56 — Phase 1.2 tier-2 sourcing decision (SPECIFY_DEFER, unanimous 3/3)

**Ballot hash:** `e85a24ec9864…`
**Result:** **SPECIFY_DEFER — 3/3**. Ratify the deterministic selection-rule design now;
**defer the build**.

The panel agreed the proposed rule (published source set; window `[now−W, now]`; order by
`(resolution_timestamp, question_id)`; oldest K unreviewed; W/K/sources as panel-gated
published constants) is sound and collapses agenda control from "pick each question" to
"pick the source set once, in public, under 2/3." But V1 surfaced the deeper requirement
that makes deferral correct:

> Determinism *at a feed snapshot* ≠ "reproducible from public inputs" over time. Two
> preconditions the rule alone can't supply: **(1)** the feed operator can edit
> timestamps / add / remove / reorder — moving the cherry-pick surface from *our* operator
> to the *feed's*; **(2)** the evaluation instant "now" is unpinned. Closing both needs
> hash-pinned, timestamped feed snapshots + a pinned evaluation instant — the **CIP-3
> external-anchor mechanism**. Building tier-2 now would deploy the rule on un-anchored,
> operator-mutable inputs, re-introducing the exact cherry-pick surface the binding
> constraint forbids.

**Deferral trigger (consensus):** build tier-2 when CIP-3's external-anchor mechanism is
live (tamper-evident feed *inputs*, not just a deterministic rule over them) **and/or** the
token/substrate phase begins (real demand turns the feed from architecture rehearsal into a
fee-bearing product dependency). Tier 1 already proves autonomous sourcing exists (round
48: existence not economics), so nothing is lost by waiting.
